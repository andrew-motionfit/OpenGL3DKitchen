# 3D Kitchen Scene Project

## Project Overview

This project is a 3D kitchen scene created using C++ and OpenGL. The scene includes kitchen objects such as stools, an island, countertops, and other environmental details. Throughout the project, I worked with shape drawing, object positioning, texturing, lighting, camera movement, and different camera views such as orthographic and perspective projection.

The purpose of this project was to demonstrate my understanding of computational graphics and visualization by creating a fully realized 3D world. The project also helped me better understand the development process behind interactive graphics applications.

---

## Design Approach

### How do I approach designing software?

I approach software design by first identifying the main goal of the project and then breaking that goal into smaller parts. For this project, I began by deciding what type of 3D environment I wanted to create. I chose a kitchen scene because it gave me the opportunity to include multiple objects, surfaces, textures, lighting effects, and camera controls in one organized space.

Instead of trying to build the entire scene at once, I focused on one object or feature at a time. I started with basic shapes and gradually built the scene by adding the island, stools, countertops, and other kitchen elements. This helped me stay organized and made the project easier to manage.

### What new design skills has this project helped me craft?

This project helped me improve my ability to think visually and spatially. I had to consider how each object should be scaled, positioned, textured, and lit so that the scene looked realistic and balanced. I also learned how important camera perspective is when designing a 3D environment.

The project also helped me understand that small design choices can have a large impact. For example, changing the position of a light source, adjusting an object’s size, or modifying a texture can completely change the appearance of the scene.

### What design process did I follow?

My design process was iterative. I started with simple objects and then added detail over time. After placing objects in the scene, I tested how they looked from different camera angles and made adjustments when something looked incorrect or unnatural.

I also used feedback from each milestone to improve the scene. As the project developed, I refined the layout, lighting, textures, and camera movement to make the final scene feel more complete.

### How can this design approach be applied in the future?

The design tactics I used in this project can be applied to future software projects by encouraging a step-by-step development process. Planning the overall structure first, building in smaller sections, testing often, and improving through iteration are useful strategies for many types of programming work.

This approach helps prevent a project from becoming overwhelming and makes it easier to identify problems early.

## Development Approach

### How do I approach developing programs?

I approach program development by building a strong foundation first and then adding complexity gradually. For this project, I started by working with the basic structure of the OpenGL program. Once I had objects appearing in the scene, I began adding transformations, textures, lighting, and camera controls.

Each part of the program needed to work together. Object placement, rendering, lighting, and camera movement all affected the final result, so I had to carefully test and adjust the code throughout development.

### What new development strategies did I use?

One important development strategy I used was working more intentionally with OpenGL’s rendering pipeline. I practiced using transformations such as scaling, rotation, and translation to control the size and position of objects in the scene.

I also worked with texture mapping to make basic shapes look more realistic. Lighting was another important strategy because it helped create depth and made the kitchen scene feel more natural. I also added camera movement and changed camera operations, including working with orthographic and perspective views.

### How did iteration factor into development?

Iteration was a major part of this project. Many parts of the scene did not look correct the first time I created them. I had to repeatedly adjust object positions, camera settings, texture coordinates, and lighting values.

Each test helped me understand what needed to change. Through this process, the scene became more polished and complete.

### How did my development approach evolve?

At the beginning of the milestones, my main focus was getting objects to appear on the screen. As I moved through the project, my focus shifted toward improving the quality, organization, and realism of the scene.

By the end of the project, I was thinking more carefully about how the code, camera, lighting, and textures worked together. My approach evolved from simply building individual objects to creating a complete and connected 3D environment.
